# Product

## Register

product

## Users

JARVIS is for David and similar high-context students or operators who need a secretary-like scheduler during real planning moments. They arrive with tasks, calendar commitments, preferences, memory, source uncertainty, and limited energy. The primary use scene for this overhaul is late-night planning on a laptop, with mobile support for triage and quick corrections.

## Product Purpose

JARVIS helps the user turn real commitments and real tasks into a practical schedule. It should preserve calendar truth, expose source failures, remember preferences, name tradeoffs, and help the user recover when plans change. The first-run value moment is a real generated plan based on real constraints, not a demo dashboard.

## Brand Personality

Calm, precise, protective. JARVIS should feel like a competent secretary quietly assembling the night before tomorrow: low-glare, direct, and steady under pressure.

## Anti-references

Avoid generic SaaS dashboards, metric-card hero patterns, decorative gradients, fake task data, chatbot-wrapper layouts, and plain Google Calendar clones. Avoid interfaces where every panel has equal weight, dead space hides the next action, or onboarding explains instead of getting the user to a real plan.

## Design Principles

- Schedule first: the timeline and next planning decision must be visually primary.
- Real state only: empty, loading, auth, and source-failure states must be honest and actionable.
- Quiet command: natural-language input is central, but it should support the schedule rather than replace it.
- Progressive setup: onboarding lives inside the product, uses real actions, and remains dismissible and replayable.
- Tradeoffs visible: when a plan compresses, defers, or risks something, the interface should make that legible.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Preserve visible focus states, keyboard-accessible controls, clear icon labels, reduced-motion compatibility, and responsive layouts for phone triage. Do not rely on color alone for task, source, or plan state.
